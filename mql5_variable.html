變數宣告與資料結構

一、基本變數宣告

MQL5語法基本承襲自C++，所以宣告變數的方式也跟C++一模一樣，都是 資料型態 變數名稱 = 變數的值 ;

MQL5目前是支援中文的變數名稱，不過有時候會因此觸發不可預期的錯誤，為了避免發生這樣的問題，還是要盡可能地取英文的變數

以下是宣告變數的範例
int 整數 = 10;
float 小數 = 10.999;
string 字串 = "我是字串";
bool 布林變數 = 整數>小數;
ulong 長整數 = 100;

對新手來說常犯的錯誤就是沒有打分號，在每一次的變數宣告或是運算結束後都應該要加上分號讓電腦知道該行的結尾在哪

上述的範例中我們宣告了五種不同資料型態的變數

分別是
整數 小數 字串 布林變數 長整數

一般來說比較有問題的是布林變數和長整數

布林變數是專門用來儲存條件判斷的資料型態，儲存的值會是true或是false

以上面的例子來說，因為整數是小於小數，所以布林變數的值便是false

而長整數跟整數的差別在於兩者可儲存的資料長度不同

有時候有可能會因為資料的大小超過變數本身能儲存的長度而造成錯誤

接下來介紹一下變數的資料轉換還有運算
int 第二個整數 = (int) 小數;
float 第二個小數 = (float)整數;
第二個整數 = 第二個整數+整數;
string 第二個字串 = DoubleToString(小數);
bool 第二個布林變數 = ((整數)>(小數) || (整數)<(小數));

前兩行示範了怎麼把變數轉成小數和整數

第三行則是加法運算，減法乘法除法也是如此

第四行是呼叫了一個DoubleToString的函式將小數轉換成字串

函式的部分會在後面的文章再做說明

布林變數的運算就是邏輯運算

邏輯運算有三種運算方式：而且(&&)、或是(||)、不是(!)三種運算方式

主要是用來做條件運算

以上圖來說，第二個布林變數的值就是 整數大於小數，或是整數小於小數也就是true

寫到這邊大家應該還是搞不懂這東西在幹嘛，不過等到下一章的邏輯判斷時應該會比較懂他的用途


二、常用的資料結構─陣列

所謂的陣列其實就是一個存放許多相同資料型態的箱子

而這個箱子中的每一個格子都有他的編號

你可以把它取出來，也可以用來存放其他的值

編號的開頭是從0開始，也就是說如果你宣告的陣列長度是3，那麼編號就是0~2

我們宣告陣列的方式為
資料型態 陣列名稱[陣列的長度] = {陣列的內容物1,陣列的內容物2,...,陣列的內容物N} ;

下圖將示範如何宣告陣列

以上圖的範例來說，array1D[0]的值就是5，array1D[2]的值則是25

陣列在MQL5中是一個非常重要的資料格式

許多資料都是以資料的形式儲存，例如指標、字串等

因此請務必要搞懂陣列的運作方式
